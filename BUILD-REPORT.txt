1. WHAT IT IS
------------------------------------------------------------------------------
A single-user, local-first AI + Colombia/Bogota news triage dashboard.
Core loop: Fetch -> Normalize -> Dedupe -> Score -> Explain -> Triage ->
Save/Ignore -> improves future ranking.

Stack (locked, no deviations):
  Python 3.12, FastAPI, Uvicorn
  SQLAlchemy 2.x + Alembic, SQLite (WAL, FK on, busy_timeout, synchronous=NORMAL)
  FTS5 full-text search (virtual table + 3 sync triggers)
  Jinja2 server-rendered HTML + HTMX 2.x (vendored, no CDN) — NO React
  httpx + feedparser (RSS), trafilatura + BeautifulSoup (clipper)
  pydantic-settings, pyyaml, filelock, pytest + respx
  Cron-driven ingest (system crontab), no in-process scheduler

3. SCORING (v1_base, deterministic — PLAN.md §6.1)
------------------------------------------------------------------------------
final_score =
    0.20 * source_score        (0.7*trust + 0.3*priority)
  + 0.20 * freshness_score      (exp(-hours_since_published / 36); None -> 0.5)
  + 0.20 * keyword_score        (saturating: 1 - exp(-sum_matched_weights/1.5))
  + 0.10 * category_score       (categories_priority[category])
  + 0.20 * user_feedback_score  (affinity maps; cold-start floor 0.3 until 50 actions)
  + 0.10 * novelty_score        (1 - max bigram-Jaccard vs recent-24h titles)
Every score carries an explanation and a score_runs audit row.

5. HOW TO RUN
------------------------------------------------------------------------------
  cd WORKSPACE/proyects/NEWSDASHBOARD
  python -m venv .venv && source .venv/bin/activate
  pip install -e ".[dev]"
  cp .env.example .env
  alembic upgrade head
  python -m scripts.seed_sources
  uvicorn app.main:app --reload          # http://localhost:8000  (docs at /docs)
  curl -X POST http://localhost:8000/ingest/run   # batch fetch + score

  pytest -q                              # 79 passing
  python -m scripts.verify_feeds         # check feed health
  python -m scripts.backup_db
  python -m scripts.recalculate_scores --all --dry-run
